# 01 이 책을 읽는 방법

- 상태: 초안 완료
- 목표 분량: 4쪽

## 목적

- 학습 모드(빠른 학습/표준 학습/심화 학습)를 제시한다.
- 챕터 구성 요소와 읽는 순서를 설명한다.
- 주간 학습 계획 예시를 제공한다.

## 먼저 알아둘 원칙

이 책은 처음부터 끝까지 순서대로 읽을 때 가장 효과적입니다.
다만 시간과 목적이 다르므로, 학습 모드를 나눠서 제시합니다.

## 학습 모드 3가지

### 1) 빠른 학습 모드 (1~2주)

- 대상: Redis를 당장 업무에 적용해야 하는 독자
- 읽기 범위: Part 01, Part 05, Part 06 우선
- 방법: 개념 1회독 + 예제 실행 + 체크리스트 확인

### 2) 표준 학습 모드 (4~6주)

- 대상: 기초부터 실무 설계까지 안정적으로 익히고 싶은 독자
- 읽기 범위: 전 파트 순서대로 진행
- 방법: 챕터별 요약 작성 + 연습문제 풀이 + 오답 정리

### 3) 심화 학습 모드 (8주 이상)

- 대상: 운영/성능/장애 대응까지 깊게 다루고 싶은 독자
- 읽기 범위: 전 파트 + 부록 + 자체 실험 기록
- 방법: 각 챕터 예제 변형 실습 + 성능 측정 리포트 작성

## 챕터를 읽는 순서

각 챕터는 아래 순서로 읽는 것을 권장합니다.

1. 학습 목표: 이번 장에서 얻을 결과를 확인한다.
2. 핵심 개념: 정의와 동작 원리를 이해한다.
3. 실습 예제: 명령어를 직접 실행한다.
4. 설계 포인트: 실무 의사결정 기준을 정리한다.
5. 자주 하는 실수: 반례를 통해 실패 패턴을 학습한다.
6. 요약/연습문제: 이해도를 점검한다.

## 실습 규칙

실습은 "입력"보다 "결과 해석"이 더 중요합니다.
아래 규칙을 지키면 학습 효율이 크게 올라갑니다.

- 명령어 실행 전: 목적을 한 줄로 먼저 기록한다.
- 명령어 실행 후: 결과와 예상이 일치하는지 확인한다.
- 예상과 다르면: 원인(키 상태, TTL, 자료형)을 추적한다.
- 위험 명령은: 로컬 환경에서만 실행한다.

## 노트 정리 방식

챕터당 한 페이지로 아래 항목을 정리해보세요.

- 오늘의 핵심 용어 3개
- 오늘의 필수 명령어 3개
- 오늘의 실수 포인트 1개
- 오늘의 실무 적용 아이디어 1개

이 기록은 나중에 면접, 코드 리뷰, 장애 회고에서 매우 유용합니다.

## 주간 학습 계획 예시 (표준 모드)

- 1주차: Part 01 (기초 개념)
- 2주차: Part 02 (자료구조)
- 3주차: Part 03 (명령/패턴)
- 4주차: Part 04 (운영/아키텍처)
- 5주차: Part 05 (성능/보안)
- 6주차: Part 06 + 부록 + 총정리

## 체크포인트

아래 질문에 "예"라고 답할 수 있으면 다음 파트로 넘어가도 좋습니다.

- 키 네이밍과 TTL을 스스로 설계할 수 있는가?
- `KEYS` 같은 위험 명령을 피해야 하는 이유를 설명할 수 있는가?
- 장애 상황에서 최소한의 복구 순서를 말할 수 있는가?
