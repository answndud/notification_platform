# Appendix A - 용어집

- 상태: 초안 완료
- 목표 분량: 2쪽

## 핵심 용어

- Redis: 메모리 기반 데이터 구조 저장소
- Key: 데이터를 찾기 위한 식별자
- Value: 키에 저장되는 실제 값
- TTL(Time To Live): 키가 살아있는 시간
- Eviction: 메모리 부족 시 키를 제거하는 정책
- RDB: 시점 스냅샷 기반 영속성 방식
- AOF: 명령 로그 기반 영속성 방식
- Replica: Primary 데이터를 복제하는 노드
- Sentinel: 장애 감지/자동 전환 관리 구성
- Cluster: 해시 슬롯 기반 분산 저장 구조
- HotKey: 트래픽이 과도하게 집중되는 키
- BigKey: 크기나 원소 수가 지나치게 큰 키
- Cache-Aside: 앱이 캐시 miss 시 원본 조회 후 캐시 저장
- Pub/Sub: 실시간 발행/구독 메시징 패턴
- Stream: 로그형 이벤트 저장/소비 자료구조

## 용어 사용 원칙

- 첫 등장은 한국어 설명 + 영문 원어 병기
- 이후는 동일 표현을 일관되게 사용
