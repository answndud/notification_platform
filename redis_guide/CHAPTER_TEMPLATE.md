# Redis 챕터 템플릿

이 문서는 `redis_guide` 챕터 품질과 형식을 통일하기 위한 작성 기준입니다.

## 파일/메타 규칙

- 파일명: `chNN_topic.md` 형식
- 상태: `초안 완료` -> `최종 교정 완료`
- 목표 분량: 기본 8~12쪽(핵심 챕터는 10~14쪽 가능)

```md
# Chapter NN - 챕터 제목

- 상태: 초안 완료
- 목표 분량: 10쪽
```

## 필수 섹션 순서

1. 학습 목표
2. 핵심 개념
3. 직관 그림(선택)
4. 실습 예제
5. 실패 시 확인 항목
6. 설계 포인트
7. 자주 하는 실수
8. 요약
9. 초보자 체크
10. 연습문제
11. 챕터 체크리스트

## 학습 목표 템플릿

- 개념 설명 가능: (핵심 개념)
- 실습 수행 가능: (핵심 명령/절차)
- 실무 판단 가능: (선택 기준/운영 기준)

## 핵심 개념 작성 규칙

- 기본 흐름: 정의 -> 예시 -> 주의점
- 첫 등장 용어는 한국어 + 영문 원어 병기
- "왜 이게 중요한지"를 3~5문장으로 명확히 설명

## 실습 예제 템플릿

실습 목적: 이 실습으로 확인할 운영/설계 포인트를 1줄로 작성합니다.

```bash
# 명령 1
# 명령 2
```

예상 결과/관찰 포인트:
- 정상 시 신호 1개
- 실패 시 신호 1개
- 실무에서의 의미 1개

## 실패 시 확인 항목 템플릿

1. 증상
- 대표 로그/오류
- 우선 점검 항목(설정/키/자료형/네트워크)

2. 증상
- 대표 로그/오류
- 우선 점검 항목

3. 증상
- 대표 로그/오류
- 우선 점검 항목

## 설계 포인트 템플릿

- 설계 원칙 3개 이내로 명확히 작성
- 선택 기준에 수치 또는 조건 포함

운영 반영 체크리스트:
- 정책 문서 반영 여부
- 알림/지표 기준 연결 여부
- 런북/회고 반영 여부

## 요약/문장 품질 규칙

- 요약은 2~4개 불릿으로 끝냄
- 한 문장은 가능한 25자 내외 유지
- 모호한 표현은 기준과 함께 작성
