## 왜 Redis를 따로 배워야 할까

많은 팀이 Redis를 "캐시"로만 기억합니다. 하지만 실무에서 Redis는 캐시를 넘어
세션 저장소, 분산 락, 랭킹, 큐, 실시간 집계 등 다양한 역할을 맡습니다.

문제는 사용 빈도에 비해 학습이 단편적으로 진행된다는 점입니다.
`GET/SET`만 알고 시작하면 초기에는 빠르게 구현되지만,
운영 단계에서 TTL 누락, 키 충돌, 메모리 급증, 장애 복구 실패 같은 문제가 반복됩니다.

이 가이드는 "기능 구현"에서 멈추지 않고
"운영 가능한 설계"까지 연결하는 것을 목표로 합니다.

## 이 책의 독자

- Redis를 처음 접하는 백엔드 개발자
- Spring Boot 기반 서비스를 운영하거나 준비하는 개발자
- 캐시/세션/토큰 저장 전략을 체계적으로 정리하고 싶은 개발자

아래 선수 지식이 있으면 좋지만, 필수는 아닙니다.

- HTTP와 REST API 기초
- 데이터베이스(SQL) 기본 개념
- Docker 명령어 기초

## 이 책이 다루는 범위

이 가이드는 다음 세 가지를 중심으로 구성됩니다.

1. 개념: Redis 내부 동작, 자료구조, 명령어 원리
2. 설계: 키 네이밍, TTL 정책, 무효화 전략, 장애 대응
3. 적용: Spring Boot 통합, 토큰/세션 설계, 운영 체크리스트

즉, "명령어를 외우는 책"이 아니라
"왜 이 명령어를 이 상황에서 쓰는지 판단하는 책"입니다.

## 이 책의 약속

이 책은 다음 약속을 지킵니다.

- 용어는 첫 등장 시 한국어 설명 + 영문 원어를 함께 제시한다.
- 예제는 가능한 한 짧게 제시하고, 결과 해석을 자세히 설명한다.
- 운영 위험 명령과 반례를 반드시 함께 다룬다.
- 각 챕터 끝에 요약과 연습문제를 넣어 학습을 고정한다.

## 300쪽 학습 여정

전체 학습은 6개 파트와 부록으로 구성됩니다.

- Part 01: Redis를 안전하게 시작하기 위한 기초
- Part 02: 자료구조별 사용 기준과 명령어 패턴
- Part 03: 트랜잭션, 스크립트, 스트림 등 실전 기능
- Part 04: 영속성, 복제, 클러스터, 운영 관측
- Part 05: 성능 병목과 보안/비용 관리
- Part 06: 프로젝트 적용 사례와 런북
- 부록: 용어집, 치트시트, 연습문제 정답

한 장씩 따라오면, 최종적으로 다음 질문에 스스로 답할 수 있어야 합니다.

- "이 데이터는 Redis에 둬야 하는가?"
- "키와 TTL은 어떤 기준으로 설계해야 하는가?"
- "장애가 발생했을 때 어떤 순서로 복구해야 하는가?"

## 학습 전 준비

- `redis-cli` 접속 환경
- 실습 기록용 노트(명령어, 결과, 관찰 포인트)

환경 준비는 Chapter 02에서 단계별로 진행합니다.

Redis는 "빠른 저장소"이기 전에 "설계 품질이 성능을 좌우하는 시스템"입니다.
