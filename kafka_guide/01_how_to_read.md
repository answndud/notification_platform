# 01 이 책을 읽는 방법

- 상태: 초안 완료
- 목표 분량: 4쪽

## 목적

- 학습 모드(빠른/표준/심화)를 제시한다.
- 챕터 구성 요소와 읽는 순서를 설명한다.
- 주간 학습 계획 예시를 제공한다.

## 학습 모드

### 빠른 학습(1~2주)
- 대상: 당장 Kafka를 도입해야 하는 독자
- 권장 범위: Part 01, Part 03, Part 06

### 표준 학습(4~6주)
- 대상: 개념부터 운영까지 안정적으로 학습하려는 독자
- 권장 범위: 전 파트 순서 학습

### 심화 학습(8주+)
- 대상: 운영 자동화/장애 대응까지 깊게 다루려는 독자
- 권장 범위: 전 파트 + 부록 + 성능 실험

## 챕터 읽기 순서

1. 학습 목표
2. 핵심 개념
3. 실습 예제
4. 설계 포인트
5. 자주 하는 실수
6. 요약/연습문제

## 실습 규칙

- 명령 실행 전 목적을 한 줄로 작성한다.
- 결과가 예상과 다르면 원인을 기록한다.
- 실패 로그를 버리지 말고 재현 절차와 함께 저장한다.
- 운영 위험 명령은 로컬/테스트에서만 실행한다.

## 체크포인트

- 파티션/오프셋/리밸런스를 설명할 수 있는가?
- 전달 보장 수준을 요구사항에 맞게 고를 수 있는가?
- 장애 시 우선순위(완화->복구->회고)를 정할 수 있는가?
