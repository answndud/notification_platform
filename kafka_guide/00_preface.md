## 왜 Kafka를 따로 배워야 할까

Kafka는 단순 메시지 큐가 아니라, 이벤트 스트리밍 플랫폼입니다.
처음에는 "메시지를 보낸다/받는다"로 보이지만,
실무에서는 파티션, 오프셋, 재시도, 순서 보장, 장애 복구가 함께 얽힙니다.

즉, Producer/Consumer 코드만 작성해서는 부족합니다.
이 가이드는 기능 구현을 넘어 운영 가능한 설계까지 다룹니다.

## 이 책의 독자

- Kafka를 처음 접하는 백엔드 개발자
- 이벤트 기반 아키텍처를 실무에 적용하려는 개발자
- Spring Boot + Kafka 운영 기준을 정리하려는 개발자

## 이 책의 범위

1. 개념: Broker/Topic/Partition/Offset/Consumer Group
2. 설계: 전달 보장, 재시도, DLQ, Outbox, 스키마 진화
3. 운영: 모니터링, 장애 대응, 보안, 용량 계획
4. 적용: Spring Kafka와 프로젝트 통합 패턴

## 학습 전 준비

- Docker 기반 Kafka 실행 환경
- CLI(`kafka-topics`, `kafka-console-producer`, `kafka-console-consumer`) 또는 `kcat`
- 실습 기록 노트(명령, 결과, 관찰 포인트)
