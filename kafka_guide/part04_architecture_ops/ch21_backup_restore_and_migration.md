# Chapter 21 - 백업/복구/마이그레이션

- 상태: 초안 완료
- 목표 분량: 11쪽

## 학습 목표
- Kafka 백업/복구 전략을 설명할 수 있다.
- 복구 리허설 절차를 설계할 수 있다.
- 마이그레이션 리스크를 줄일 수 있다.

## 핵심 개념

Kafka에서 백업은 단순 파일 복사가 아니라
"재처리 가능한 보관 전략 + 복구 절차"로 접근해야 합니다.

## 실습 예제

```text
백업 대상: 로그 데이터, 설정, 토픽 정책, 운영 런북
복구 점검: 토픽 복원 -> 소비 그룹 동작 -> 핵심 API 검증
```

## 설계 포인트
- RPO/RTO를 먼저 정의한다.
- 복구 성공 기준을 수치로 명시한다.
- 마이그레이션은 증분/롤백 계획을 포함한다.

## 자주 하는 실수
1. 백업은 있는데 복구 테스트가 없음
2. 마이그레이션 범위/순서 미정의
3. cutover 실패 시 롤백 계획 부재

## 요약
- 백업의 목적은 복구 성공이다.
- 마이그레이션은 체크리스트 기반으로 진행해야 한다.

## 연습문제
### 기초
1. 백업 정책 표준 항목을 작성해보세요.
2. 복구 테스트 항목 10개를 작성해보세요.

### 응용
1. 토픽 이전 cutover 계획을 작성해보세요.
2. 실패 시 롤백 절차를 설계해보세요.
