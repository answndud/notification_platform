# 한국어 집필 스타일 가이드

## 기본 원칙

- 대상 독자는 Kafka 비전공 초급 개발자입니다.
- 한 문단에는 핵심 메시지 하나만 담습니다.
- "정의 -> 예시 -> 주의점" 순서를 기본 골격으로 사용합니다.

## 용어 표기 규칙

- 첫 등장: 한국어 설명 + 영문 원어 병기
  - 예: 소비자 그룹(Consumer Group)
- 두 번째 등장부터는 한국어 또는 약어를 일관되게 사용합니다.
- 줄임말만 단독으로 시작하지 않습니다.

## 문장 규칙

- 가능하면 한 문장을 25자 내외로 유지합니다.
- 모호한 표현("적절히", "상황에 따라")은 기준과 함께 제시합니다.
- 명령형보다 설명형 문체를 기본으로 사용합니다.

## 예제 규칙

- 모든 명령어 예제는 실행 목적을 먼저 적습니다.
- 예제 아래에는 예상 결과 또는 해석 포인트를 반드시 추가합니다.
- 운영 위험 명령은 경고 문구를 함께 표시합니다.
